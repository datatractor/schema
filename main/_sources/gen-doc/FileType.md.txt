| ---  | --- | --- | --- |
| [id](id.md) | 1..1 <br/> [String](String.md) | A unique identifier for the entry within the MaRDA registry namespace, this<br/>should be a shorthand label rather than a UUID. Only lower-case alphanumeric and<br/>dash ("-") characters are permitted. | direct |
| [description](description.md) | 1..1 <br/> [String](String.md) | A human-readable outline of the entry, its format, data content and uses. | direct |
| [name](name.md) | 1..1 <br/> [String](String.md) | A recognisable name for the entry. | direct |
| [subject](subject.md) | 0..* <br/> [String](String.md) | Any keywords, phrases or classification codes that are relevant to the entry,<br/>e.g., particular scientific domains of applicability, or experimental<br/>techniques. | direct |
| [associated_vendors](associated_vendors.md) | 0..* <br/> [String](String.md) | If this file type is associated with particular software or instrument vendors,<br/>they can be listed here. | direct |
| [associated_instruments](associated_instruments.md) | 0..* <br/> [String](String.md) | Any instrument, or class of instruments, that typically create the data encoded<br/>into such files. | direct |
| [associated_software](associated_software.md) | 0..* <br/> [String](String.md) | Any software (proprietary or otherwise) that produces such files. | direct |
| [base_formats](base_formats.md) | 0..* <br/> [FileType](FileType.md) | If this format uses any particular underlying generic formats, e.g., CSV, JSON,<br/>HDF5, then they can be listed here. | direct |
| [associated_standards](associated_standards.md) | 0..* <br/> [String](String.md) | If this is a subformat that follows a particular well-defined standard, e.g.,<br/>CIF, NeXus, then it can be listed here. | direct |
| [registered_extractors](registered_extractors.md) | 0..* <br/> [String](String.md) | A list of MaRDA extractor IDs that support this file type. This field should be<br/>auto-populated by a registry. | direct |
